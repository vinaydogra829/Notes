Paired And Unpaired Tag

    What is Tag ?
     - Text palced between '<' and '>' is called as tag
	Eg <html>
	
    Tags are classified into two types
	    1. Paired tags
	    2. Non-Paired tags
	
     1. Paired tags
	    - The tags that have both opening and closing
	      tags are called as paried tags.
	  Eg	
		<html>
			...
			...
			...
		</html>
		
		<body>
			...
			...
			...
		</body>
	Note:- The closing tags start with '/'


     2. Non-Paired
	    - The tags that have only opening tag but no closing
	       tag are called as non-paired tags.
	    - These tags are also called as 
		   i) Self closed tags
		   ii)Forcefully closed tags
		   iii)Empty tags
		   iv)Singular tags.
		
	    Eg	<br>	or 	<br/>	Break tag(line break)
		 <hr>	or 	<hr/>	Hrizontal rule (h. line)
		 	or		image tag

Color Codes 

- HTML colors are defined in 24 bit RGB values
- These values are represented in hexadecimal form
- Value start with #
- 0 is the lowest value
- F is the highest value

   Example: R   G   B

       # FF FF FF -> White #FFFFFF

       # 00 00 00 -> Black #000000

       # FF 00 00 -> Red #FF0000

       # 00 FF 00 -> Green #00FF00

       # FF FF FF -> Blue #0000FF

GET Method

    -This method is not secure for data.
    -This method is used to send only limited data (2MB).
    -It can carry raw data from client to server.
    -Fastest response method.
    -Features of Get Method
    -Get requests can be cached.
    -Get requests remain in browser history
    -Get requests should not deal with sensitive data.
    -Get requests' length is restricted.
    -Get requests have excellent performance.
    -Get requests can not upload images.

    Eg:                     

    <!doctype html>
    <html>
    <head>
    </head>
    <body>
    <form Method = "GET" action = "https://www.google.com" name = "my_form>
    <label>Username:- </label><input type = "text" name = "uname" placeholder = "Enter Username">
    <label>Password:- </label><input type = "password" name = "upwd" placeholder = "Enter Password">
    <input type = "submit" value = "Login">
    input type = "Reset" value = "Clear">
    </form>
    </body>
    </html>

POST Method

    -This method provides security for data.
    -This method can send bulk of data.
    -It can carry encrypted data from client to server.
    -Fastest response method.
    -Features of Get Method
    -Post requests can not be cached.
    -Post requests do not remain in browser history
    -Post requests can deal with sensitive data.
    -Post requests have no limit for length of data.
    -Post requests have good performance.
    -Post requests can upload images.
    -Post requests can not be bookmarked.
    -HTTP Status Messages
            1xx -> 100 to 199 : Informational
            2xx -> 200 to 299 : Successful
            3xx -> 300 to 399 : Redirection
            4xx -> 400 to 499 : Client Side Error
            5xx -> 500 to 599 : Server Side Error
